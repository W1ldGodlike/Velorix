# URL на вкладке «Обработка» (§7.4)

Блок **«URL → скачивание → кодирование»** над очередью файлов связывает yt-dlp и ffmpeg без переключения на вкладку «Скачивание».

## Как пользоваться

1. Вставьте ссылку (используется **первая непустая строка** поля).
2. **Сначала скачать через yt-dlp** — при включении параметры совпадают с **текущими полями вкладки «Скачивание»**: качество, контейнер, папка вывода (`OutputDir`), прокси, cookies, дополнительные аргументы эксперта и т.д. Если галочку снять, в очередь попадает только прямой **http(s)** URL для ffmpeg (подходит для прямых ссылок на файл).
3. **Затем запустить пакет кодирования** — после постановки файлов в очередь сразу стартует ffmpeg с текущими опциями и параллелизмом.

## История

Записи вида `processing+yt-dlp` можно **повторить** кнопкой «Повтор»: приложение откроет вкладку обработки и подставит URL в это поле.

## Анализ ffprobe

Если в очереди первым стоит удалённый http(s) вход, кнопка **Анализировать (ffprobe)** и пункт контекстного меню строки по-прежнему вызывают ffprobe для этого URL (на практике работает не для всех источников).
