# Аппаратное ускорение (NVENC, AMF, QSV, VAAPI)

В редакторе уже доступны **аппаратные** видеокодеки в правой панели «Формат» и режимы **hw_auto** / **hw_auto_hevc** (автовыбор по результату probe). Строка **«Обработка: …»** внизу окна и подсказка у поля кодека показывают, какой кодер выбран, модель GPU (NVIDIA через nvidia-smi) и лимиты NVENC.

## Быстрый старт

1. Откройте короткий клип H.264 (10–30 с) в редакторе.
2. Дождитесь probe в rail «Формат» — в списке кодеков появятся доступные HW-варианты.
3. Для проверки на своей видеокарте: **Настройки → Зависимости → Ручной smoke NVENC / VAAPI** — пошаговый чеклист и кнопка копирования (тот же текст попадает в Support ZIP).

## Режимы

| Режим                       | Когда использовать                                              |
| --------------------------- | --------------------------------------------------------------- |
| Ручной HW-кодек             | Нужен конкретный NVENC / AMF / QSV / VAAPI                      |
| **hw_auto**                 | Пусть приложение выберет лучший доступный HW-кодек при экспорте |
| Программный (libx264 и др.) | Максимальная совместимость и тонкая настройка CRF               |

Панель **«Оценить»** (15 с) сравнивает кандидатов по скорости и нагрузке CPU/GPU.

## Платформы

- **Windows + NVIDIA** — NVENC (см. чеклист в настройках).
- **Linux** — VAAPI (Intel/AMD), при наличии `/dev/dri` и vaapi в `ffmpeg -hwaccels`.
- **macOS** — VideoToolbox в списке кодеков; ручной smoke в настройках ориентирован на Win/Linux.

См. также [ffmpeg-rail-presets.md](ffmpeg-rail-presets.md) и [faq-troubleshooting.md](faq-troubleshooting.md).
