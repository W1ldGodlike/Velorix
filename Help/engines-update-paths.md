# Инструменты: где лежат и как обновить

## Встроенная копия

В установленной программе рядом лежат проверенные версии **загрузчика ссылок** и **набор для перекодирования**. Если они на месте, можно работать сразу после установки.

## Своя папка

При необходимости путь к каждому инструменту можно указать вручную в настройках. Это удобно для тестовых сборок или корпоративных политик.

## Кнопка «Скачать»

Если встроенных файлов нет, статусная строка предложит **скачать** нужный компонент в служебную папку пользователя. Прогресс показывается внизу окна.

## Проверка целостности

Для сборки под Windows можно включить строгую проверку контрольных сумм — это защищает от подмены бинарников. Подробности для сопровождения — в документации сборки в репозитории проекта.

## Packaged smoke (§19 / §21)

После `npm run pack:dir` (Windows) или `pack:linux:dir` / `pack:mac:dir` проверьте бинарники в `dist/*-unpacked` (`smoke:packaged-*` в CI).
