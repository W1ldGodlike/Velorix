Ты работаешь в репозитории FluxAlloy (Electron + TypeScript + React).

Задача этой итерации:
1. Сначала прочитай `scripts/cursor-automation/prompts/agent-contract.txt` и следуй ему как обязательной инструкции для этой и следующих итераций.
2. Открой `IMPLEMENTATION_CHECKLIST.md`: сначала блок `## Ближайший TODO спринта`, затем только связанный тематический §.
3. Открой последние 30–50 строк `IMPLEMENTATION_JOURNAL.md`: определи последний `J-NNN` и проверь, что не дублируешь уже сделанное.
4. `docs/UX_REFERENCE_V0.md` читай только если выбранный блок реально касается UI/layout/v0-паритета.
5. Реализуй крупным осмысленным куском. Не трогай FLUXALLOY_TZ.md без явной просьбы пользователя в прошлой переписке.
6. В конце выполни `npm run check` из корня репозитория; при ошибках исправь.
7. При необходимости обнови только эти два файла: IMPLEMENTATION_CHECKLIST.md (статусы; `## Ближайший TODO спринта` держи коротким — 3-7 направлений, не архив/changelog) и IMPLEMENTATION_JOURNAL.md (одна строка с номером `J-NNN`, датой/временем, источником `[SDK]` и сутью — без простыней).
   ВАЖНО: время для IMPLEMENTATION_JOURNAL.md нельзя придумывать. Перед записью обязательно выполни команду получения текущего локального времени (`Get-Date -Format "yyyy-MM-dd HH:mm:ss"` в PowerShell или эквивалент) и используй ровно её результат.
   ВАЖНО: ты SDK-агент, поэтому в журнале пиши только `- [J-NNN] ... [SDK]: ...`, никогда `[Assistant]`. `[Assistant]` — только для обычного чата с пользователем.
8. Если всё ок — один коммит с понятным сообщением на русском.

Отчитайся кратко: что сделано, что блокирует следующий шаг (если есть).
